Проект 3: Risk Management & API Integration (Crisis Case)

Кейс по управлению техническими рисками и предотвращению срыва сроков (Deadlines).

📌 Intro

Кейс по выводу проекта из критической ситуации: задержка интеграции с банковским API на 10 рабочих дней угрожала срывом запуска MVP к высокому сезону (новогодние траты).

🧪 Анализ рисков

Инструментарий: Мониторинг через Burndown Chart выявил резкое падение Velocity во втором спринте.

Root Cause: Выявлен блокер — отсутствие доступа к sandbox-среде и несогласованность payload-форматов.

🔧 Corrective Measures

Fallback-сценарий: Внедрена альтернативная фича — ручной импорт CSV вместо автоматического API (сохранение ценности для юзера без срыва срока).

Коммуникация: Назначен Integration Owner для синхронизации смежных команд.

QA-буфер: Выделены дополнительные ресурсы на тестирование обоих сценариев импорта.

📈 Результаты

Срок: Проект запущен в срок (дедлайн соблюден).

Бизнес-метрики: Сохранены KPI по вовлеченности (Adoption) за счет наличия базового функционала в пиковый период.
